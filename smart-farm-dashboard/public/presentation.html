MỞ ĐẦU

Lý do chọn đề tài & Bối cảnh

⚠️ Vấn đề thực tế

  • Nông nghiệp VN: 70% lao động nhưng năng suất thấp
  • Tưới tiêu thủ công: lãng phí 30-50% nước
  • Phụ thuộc kinh nghiệm cá nhân, thiếu dữ liệu khách quan
  • Biến đổi khí hậu → nhu cầu canh tác chính xác

💡 Giải pháp đề xuất

  • Hệ thống IoT giám sát real-time 4 thông số môi trường
  • Tích hợp AI biên (TinyML) quyết định tưới tiêu ngay trên chip
  • Hoạt động offline — không phụ thuộc Internet
  • Quản lý đa nền tảng: Web + Mobile

🎯 Mục tiêu cốt lõi

Xây dựng hệ thống IoT hoàn chỉnh End-to-End từ phần cứng (ESP32 + cảm biến + TinyML) → Backend (Node.js + MongoDB + MQTT) → Frontend (ReactJS + Flutter), cho phép tự động hóa tưới tiêu thông minh dựa trên dữ liệu cảm biến thời gian thực.

CHƯƠNG 1 — PHẦN CỨNG

Kiến trúc phần cứng hai node ESP32

🟢 Node Cảm Biến — ESP32 WROOM-32

CPU Dual-core Xtensa LX6 @ 240MHz
Nhiệt độ/Độ ẩm AHT20 (I²C) — ±0.3°C / ±2%RH
Ánh sáng BH1750 (I²C) — 1–65535 lux
Áp suất BMP280 (I²C) — bổ trợ
Độ ẩm đất Capacitive v1.2 — ADC Pin 34
TinyML TFLite Micro — model.h (~2.4KB)
Kết nối WiFi STA + SoftAP Fallback

🔵 Node Điều Khiển — ESP32-S3

CPU Dual-core Xtensa LX7 @ 240MHz
Relay 3 kênh Quạt (Pin 16), Đèn (17), Bơm (18)
Nút bấm 4 nút: Fan, Light, Pump, Mode
Màn hình OLED SSD1306 128×64 (I²C)
Chế độ Manual / Auto / Scheduled
Kết nối MQTT Subscribe + TCP Client
An toàn Debounce 50ms, Override timeout 5min
🟢 WROOM
Sensor + AI ── WiFi/MQTT ──→ ☁️ Server
Safety Windows ── MQTT ──→ 🔵 ESP32-S3
Relay Control

Khi mất WiFi: WROOM phát SoftAP → S3 kết nối TCP:8080 → Hoạt động Offline

CHƯƠNG 1 — TINYML

Trí tuệ nhân tạo biên — TensorFlow Lite Micro

🧠 Kiến trúc mô hình

Input(4) → Dense(8, ReLU) → Dense(3, Sigmoid) // 4 đầu vào (chuẩn hóa Min-Max): temp ∈ [15°C, 45°C] hum ∈ [0%, 100%] soil ∈ [0%, 100%] lux ∈ [0, 5000] // 3 đầu ra (xác suất 0-1): fan → Quạt thông gió light → Đèn chiếu sáng pump → Máy bơm nước

⚡ Thông số kỹ thuật

2.4 KB
Model Size
8 KB
Tensor Arena
INT8
Quantization
3 Ops
FC · ReLU · Sigmoid

🔄 Kỹ thuật Hysteresis

Chống bật/tắt liên tục (flapping) ở vùng biên ngưỡng:

ON threshold = 0.60 // Bật khi xác suất ≥ 60% OFF threshold = 0.45 // Tắt khi xác suất ≤ 45% // Vùng chết [0.45–0.60]: giữ nguyên trạng thái
CHƯƠNG 1 — HUẤN LUYỆN MÔ HÌNH

Quy trình huấn luyện & triển khai TinyML

📊 Dataset
10,000 mẫu CSV → 🔀 Stratified Split
80% Train / 20% Test → 📏 Min-Max Norm
4 features → 🧠 Keras Train
Adam, BCE, ES → 📦 TFLite Convert
INT8 Quantize → 📄 model.h
C byte array → 🔌 Flash ESP32

📊 Tập dữ liệu

Thuộc tính Giá trị
Tổng mẫu 10,000 mẫu
Features temp, hum, soil, lux (4 cột)
Labels fan, light, pump (3 cột nhị phân)
Train / Test 8,000 / 2,000 (stratified)

⚙️ Cấu hình huấn luyện

Tham số Giá trị
Optimizer Adam (lr=0.001)
Loss Binary Crossentropy
Epochs 100 (EarlyStopping patience=15)
Batch Size 32
Quantize Full INT8, 128 representative samples

🛡️ Yêu cầu phi chức năng (6 NFR)

  • Hiệu năng: Độ trễ End-to-End < 2 giây
  • Offline: Fallback TCP/SoftAP
  • Bảo mật: bcrypt + JWT + Helmet
  • AI nhỏ gọn: Model ≤ 2.4KB, Arena ≤ 8KB
  • Mở rộng: API-first, thêm device linh hoạt
  • Đa nền tảng: Web + iOS + Android

CHƯƠNG 4 — KẾT LUẬN

Đánh giá ưu / nhược điểm

✅ Ưu điểm

  • Edge AI: Quyết định tưới tiêu ngay trên ESP32, không cần Internet
  • Kiến trúc 2 node: Tách cảm biến & điều khiển — dễ bảo trì

  • Real-time: SSE push, MQTT QoS 1

⚠️ Nhược điểm & Hạn chế

  • Dữ liệu huấn luyện synthetic — chưa phản ánh hoàn toàn thực tế
  • Mô hình AI đơn giản (2 lớp) — chưa xử lý bài toán phức tạp
  • Chưa có OTA update firmware từ xa
  • MQTT chưa dùng TLS/SSL encryption
  • Chưa tích hợp camera giám sát hình ảnh
  • Chưa có Unit Test tự động cho Backend/Frontend
  • Prototype breadboard — chưa thiết kế PCB
